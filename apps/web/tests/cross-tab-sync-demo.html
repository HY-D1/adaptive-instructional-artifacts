🔧 Implementation Details

The sync mechanism uses the "set then remove" technique:

  1. When preview mode changes, we call broadcastSync(key, value)
  2. This sets a special sync channel key in localStorage
  3. The StorageEvent fires in all other tabs
  4. We immediately remove the key to avoid persisting sync messages
  5. Other tabs receive the event and update their state

Key Features

  • ✅ No infinite loops: Tabs only respond to external changes
  • ✅ Flood protection: Events older than 5 seconds are ignored
  • ✅ Error handling: Invalid events are gracefully ignored
  • ✅ Memory efficient: Sync messages aren't persisted
