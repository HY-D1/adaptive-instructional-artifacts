## Overview

This document describes the deterministic E2E authentication system for deployed preview testing. The system eliminates the need for manual credential management by providing automatic test account provisioning.

---

## The Problem (Previously)

Deployed auth-backed E2E tests were blocked by missing deterministic credentials:

```bash
# Required but not available:
E2E_INSTRUCTOR_EMAIL
E2E_INSTRUCTOR_PASSWORD
E2E_STUDENT_CLASS_CODE
```

This created **WS5-BLOCKER-001**: automated auth tests could not run against deployed previews.

## How It Works

### 1. Test-Seed Endpoint (Backend)

**File**: `apps/server/src/routes/auth.ts`

```typescript
POST /api/auth/test-seed
Headers:
  x-e2e-test-seed-secret: <secret>
```

**Security Controls**:
- Only available in `preview` and `development` environments
- Requires `E2E_TEST_SEED_SECRET` environment variable
- Returns 403 in production
- Idempotent (safe to call multiple times)

**Deterministic Credentials**:
```typescript
instructor: {
  email: 'e2e-instructor@sql-adapt.test',
  password: 'E2eTestPass!123',
  sectionCode: '<auto-generated>'
}
student: {
  email: 'e2e-student@sql-adapt.test',
  password: 'E2eTestPass!123',
  classCode: '<same-as-section-code>'
}
```

### 2. Auth Setup Integration

**File**: `tests/e2e/setup/auth.setup.ts`

The setup now:
1. Attempts to call `/api/auth/test-seed` before requiring env vars
2. Uses returned credentials if available
3. Falls back to manual provisioning if test-seed unavailable
4. Creates auth state files (`student.json`, `instructor.json`)

### 3. Environment Configuration

| Variable | Required | Default | Purpose |
|----------|----------|---------|---------|
| `E2E_TEST_SEED_SECRET` | No | `sql-adapt-e2e-test-secret` | Secret for test-seed endpoint |
| `E2E_ALLOW_TEST_SEED` | No | `true` | Set to `false` to disable test-seed |

## Usage

### Running Against Preview (Automatic)

```bash
# Set the test seed secret (or use default)
export E2E_TEST_SEED_SECRET="sql-adapt-e2e-test-secret"

# Also set on preview backend via Vercel dashboard:
# E2E_TEST_SEED_SECRET = sql-adapt-e2e-test-secret

# Run auth setup
PLAYWRIGHT_BASE_URL="https://<preview>.vercel.app" \
PLAYWRIGHT_API_BASE_URL="https://<preview-backend>.vercel.app" \
npx playwright test -c playwright.config.ts --project=setup:auth
```

### Running Against Preview (Manual Override)

If you need specific credentials:

```bash
# Disable test-seed
export E2E_ALLOW_TEST_SEED="false"

# Provide manual credentials
export E2E_INSTRUCTOR_EMAIL="your-instructor@example.com"
export E2E_INSTRUCTOR_PASSWORD="YourPassword123!"
export E2E_STUDENT_CLASS_CODE="YOUR-CLASS-CODE"

# Run auth setup
PLAYWRIGHT_BASE_URL="https://<preview>.vercel.app" \
npx playwright test -c playwright.config.ts --project=setup:auth
```

## Security Considerations

### Preview-Only Safety

The test-seed endpoint has multiple guardrails:

1. **Environment Check**: Only responds in `preview`/`development`
2. **Secret Required**: Must provide matching `x-e2e-test-seed-secret` header
3. **No Production Data**: Uses isolated preview database (ENV-001)
4. **Idempotent**: Multiple calls don't create duplicate accounts

### Secrets Management

| Environment | Secret Source | Recommended Value |
|-------------|---------------|-------------------|
| Preview | Vercel env var | `sql-adapt-e2e-test-secret` |
| CI/CD | GitHub secret | Rotate regularly |
| Local | `.env.local` | Any string |

## Troubleshooting

### "Test seed endpoint unavailable"

**Cause**: Endpoint returned error or network failure  
**Fix**: 
1. Verify `E2E_TEST_SEED_SECRET` is set on preview backend
2. Check that backend is using Neon database (not SQLite)
3. Verify environment is `preview` or `development`

### "Invalid or missing test seed secret"

**Cause**: Secret mismatch between client and server  
**Fix**: Align `E2E_TEST_SEED_SECRET` values

### "Test seed requires Neon database"

**Cause**: Backend using SQLite mode  
**Fix**: Set `DATABASE_URL` to Neon connection string

## Migration from Manual Credentials

If you previously used manual E2E credentials:

1. **Remove from env** (optional):
   - `E2E_INSTRUCTOR_EMAIL`
   - `E2E_INSTRUCTOR_PASSWORD`
   - `E2E_STUDENT_CLASS_CODE`

2. **Add to preview backend**:
   - `E2E_TEST_SEED_SECRET=sql-adapt-e2e-test-secret`

3. **Run tests**:
   ```bash
   npm run test:e2e:setup-auth:deployed
   ```
